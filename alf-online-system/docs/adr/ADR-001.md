# ADR-001: Local-First Hybrid Architecture

## Context
Home healthcare assistants need low-latency interactions, offline fallback capability, and local data ownership for safety and legal compliance.

## Decision
We implement a hybrid architecture:
* Keep all application business logic, databases (SQLite + ChromaDB), and frontends running locally on the user machine.
* Use Cloud APIs only for heavy AI model processing (LLM, Embeddings, STT).

## Consequences
* High data privacy: user data remains on-disk.
* Offline mode works for emergencies and local schedules.
* Dependency on internet connection for LLM processing.
