# ADR-002: Client-Side PHI De-identification

## Context
Sending raw patient names, medical histories, and locations to external cloud providers (Groq, Google, OpenAI) poses severe privacy risks and violates HIPAA regulations.

## Decision
All messages must pass through a local `Pseudonymizer` module before being sent to external APIs. Names, locations, and contact info are replaced with randomized placeholders, then mapped back on return.

## Consequences
* Protects patient data leakage to external models.
* Minor processing overhead (negligible).
