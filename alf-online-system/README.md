# 🏥 رفيق (Rafiq System) – المساعد الطبي والنفسي الذكي

## نظرة عامة
**رفيق** هو مساعد صوتي وتفاعلي متكامل يُقدم دعمًا طبيًا ونفسيًا مخصصًا للمستخدمين. صُمم المشروع ليعمل كنظام رعاية ذكي متكامل للمرضى في المنازل مع دعم كامل للغة العربية والإنجليزية.

يتميز رفيق بالهيكل البرمجي النظيف، والتعافي الذاتي من الأعطال، وحماية خصوصية بيانات المرضى، وفحص التداخلات الدوائية والتنبيهات، إضافةً إلى معالجة ذكية للغة الطبيعية عبر وكلاء LangGraph الذكاء الاصطناعي مع دعم RAG من منظمة الصحة العالمية (WHO).

---

## الميزات الرئيسية
1. **التفاعل الصوتي الكامل (STT/TTS)**:
   - التعرف على الصوت باستخدام نموذج Whisper عبر Groq Cloud.
   - تحويل النص إلى كلام ذو نبرة طبيعية وديناميكية باستخدام Azure Edge TTS.
2. **الذاكرة السريرية والتشغيلية (Clinical Memory)**:
   - مبني بالكامل على ChromaDB لتسجيل تفاصيل المرضى والأدوية والجرعات والتذكيرات.
   - استرجاع دلالي سريع للحقائق مع حماية صارمة للمعلومات الصحية المحمية (PHI).
3. **السلامة والأمان الطبي (Medical Guardrails)**:
   - نظام توجيه طبي (Medical Router) يفرز الأسئلة إلى عامة/طبية/طوارئ.
   - التحقق من التداخلات الدوائية عبر قاعدة بيانات RxNav المدمجة.
   - حواجز أمان لمنع التشخيص الذاتي أو تحديد الجرعات العشوائية.
4. **نظام الطوارئ (Emergency System)**:
   - إطلاق إنذار فوري بصوت عالٍ جداً وتوجيه المريض لأقرب رعاية صحية في الحالات الطارئة.

## هيكل المشروع (Clean Architecture)

```text
rafiq-system/
├── .env                  # متغيرات البيئة ومفاتيح API (محلية فقط)
├── README.md             # دليل التشغيل والاستخدام الرئيسي
├── requirements.txt      # التبعيات الأساسية والاختيارية للمشروع
├── run_rafiq.bat         # ملف تشغيل النظام كاملاً بنقرة واحدة
├── data/
│   └── drug_interactions.json   # قاعدة بيانات التداخلات الدوائية
├── docs/
│   ├── ONBOARDING.md
│   └── SECURITY.md
├── logs/
│   └── rafiq.log         # ملف السجلات العام للنظام
├── rafiq-gui/            # واجهة المستخدم الرسومية (React + Electron)
└── src/                  # الكود البرمجي للمحرك الخلفي (Python Backend)
    ├── __init__.py
    ├── gui_bridge.py     # خادم SSE و API للربط مع Electron
    ├── config/           # إعدادات النظام والتهيئة Lazy Loading
    ├── core/             # الموجه الطبي، حواجز الأمان، ووكلاء الذكاء الاصطناعي
    ├── database/         # قاعدة بيانات Chroma والذاكرة السريرية
    ├── services/         # خدمات الصوت (STT/TTS)، الجدولة، والويب الموثوق
    └── utils/            # وظائف المساعدة العامة والمعالجة اللغوية
```

## متطلبات التشغيل والتثبيت

### 1. المتطلبات البرمجية
- **Python**: إصدار `>= 3.10` (تم الاختبار على 3.12).
- **Node.js**: إصدار `>= 18` (لتشغيل واجهة Electron).
- أدوات النظام: تأكد من تثبيت `ffmpeg` وإضافة `ffplay` إلى PATH لتشغيل الأصوات.

### 2. التثبيت
قم بتثبيت التبعيات للمحرك الخلفي:
```bash
pip install -r requirements.txt
```

قم بإعداد ملف التهيئة `.env` في المجلد الرئيسي بناءً على `.env.example`:
```env
GROQ_API_KEY=your_groq_api_key_here
GOOGLE_API_KEY=your_gemini_api_key_here
```

### 3. إعداد الواجهة الرسومية (GUI)
انتقل إلى مجلد الواجهة وثبت التبعيات:
```bash
cd rafiq-gui/rafiq-gui
npm install
```

## تشغيل النظام

لتشغيل رفيق كاملاً (المحرك الخلفي مع الواجهة الرسومية) بنقرة واحدة، قم بتشغيل الملف التنفيذي:
```bash
run_rafiq.bat
```

أو يمكنك تشغيل المكونات يدوياً:

* **تشغيل المحرك الخلفي (Python Backend)**:
  ```bash
  python -m src.gui_bridge
  ```

* **تشغيل واجهة Electron (React Frontend)**:
  ```bash
  cd rafiq-gui/rafiq-gui
  npm run electron:dev
  ```

- يتم إلغاء تحديد الهوية وحماية خصوصية بيانات المريض (PHI) محلياً قبل إرسال أي نصوص لواجهات LLM الخارجية.
